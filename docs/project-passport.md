# Паспорт проекта: Салон красоты "BeautySpace"

## Предметная область
Веб-приложение для автоматизации работы современного салона красоты. Система позволяет клиентам ознакомиться с услугами и мастерами, а также записаться на процедуру онлайн без звонка администратору.

## Проблема
Клиенты тратят время на звонки, не видят свободные "окна" мастеров в реальном времени. Администратор ведет записи в бумажном журнале, что приводит к ошибкам и потере данных.

## Цель проекта
Разработать веб-интерфейс для выбора бьюти-услуг и записи к мастерам, а также панель управления для администратора салона.

## Целевая аудитория
* **Клиенты:** люди, ценящие время и предпочитающие онлайн-запись.
* **Администраторы:** сотрудники, отвечающие за график и подтверждение визитов.

## Роли пользователей
1. **Гость:** может видеть главную страницу, прайс-лист и список мастеров.
2. **Пользователь:** авторизованный клиент. Может записываться на услуги и видеть историю своих посещений.
3. **Администратор:** управляет контентом (услуги, мастера) и статусами записей.

## Контентная сущность
**Услуга** (Наименование, категория, цена, описание).
*Дополнительно:* **Мастер** (ФИО, специализация, рейтинг).

## Операционная сущность
**Запись на услугу** (ID клиента, ID услуги, ID мастера, дата, время, статус).

## Основные функции
### Гость
* Регистрация и вход.
* Просмотр каталога услуг.
* Поиск мастера по специализации.

### Пользователь
* Выбор времени и создание записи.
* Просмотр активных записей в личном кабинете.
* Отмена записи.

### Администратор
* Добавление/удаление услуг и мастеров.
* Просмотр всех записей на сегодня/неделю.
* Смена статуса записи (Принят / Выполнен / Отменен).

## Данные, которые необходимо хранить
* Таблица пользователей (логины, хэши паролей).
* Таблица услуг.
* Таблица мастеров.
* Таблица заявок (записей).

## Планируемые страницы
1. `index.html` — Главная страница.
2. `catalog.html` — Список всех услуг.
3. `profile.html` — Личный кабинет пользователя.
4. `booking.html` — Форма выбора даты и времени.
5. `login.html` / `register.html` — Вход и регистрация.
6. `admin.html` — Панель управления записями.

## Предполагаемое дизайн-направление
Светлая тема, минимализм. Основные цвета: белый, нежно-розовый (акцент), темно-серый (текст). Использование качественных фото интерьера и работ мастеров.

## Ожидаемый результат
Готовое веб-приложение с базой данных, позволяющее пройти путь от регистрации до подтвержденной администратором записи.
